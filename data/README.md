# data/ — runtime data (gitignored contents)

These are the plain files the architecture contract is built around. Contents
are **gitignored**; only the folder structure is tracked (via `.gitkeep`).

```
data/
├── incoming/        # ingest channels DROP raw files here
├── photos/          # the processor WRITES normalized photos here; the display READS here
├── thumbs/          # the processor WRITES gallery thumbnails here; the display READS here
└── manifest.json    # generated by the processor; describes the slideshow + gallery set
```

- **`incoming/`** — written and then processed inline by the `frame-uploader`
  HAOS add-on (the sole ingest channel today; the contract is channel-agnostic
  so more could be added later). There's no separate processor watching this
  folder anymore — processing happens immediately after each write.
- **`photos/`** — written **only** by the processor, read **only** by the
  display (`frame/`).
- **`thumbs/`** — written **only** by the processor (a small <=300px copy of
  each photo, filename `<id>.jpg` matching the full-size photo's id), read
  **only** by the display's gallery grid (loading thumbnails instead of
  full-size photos keeps the grid smooth on the frame's ancient WebView).
- **`manifest.json`** — generated **only** by the processor, read **only** by
  the display. Each entry also carries `favourite` (bool, default `false`,
  flipped via the add-on's `POST /favourite`) and `thumb` (the filename in
  `thumbs/`).

Nothing here is committed. See [`../CLAUDE.md`](../CLAUDE.md) for the contract.
